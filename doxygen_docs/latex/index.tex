Plauderkiste ist eine eigenständige Peer-\/to-\/\+Peer-\/\+Chatsoftware, die speziell für lokale Netzwerke (z.\+B. Uni, WG, Büro) entwickelt wurde. Plauderkiste ermöglicht es, direkt und ohne zentralen Server mit anderen Nutzern zu chatten und Bilder zu teilen. Das System ist robust, einfach zu bedienen und benötigt keinerlei Abhängigkeiten.\hypertarget{index_autotoc_md3}{}\doxysubsection{\texorpdfstring{Hauptfunktionen}{Hauptfunktionen}}\label{index_autotoc_md3}

\begin{DoxyItemize}
\item {\bfseries{Direkte Textnachrichten}} zwischen allen verbundenen Nutzern
\item {\bfseries{Bildversand}} mit Dateigrößenanzeige und Speicherung in Benutzerordnern
\item {\bfseries{Automatisches Discovery}}\+: Neue Nutzer werden im Netzwerk ohne zentrale Verwaltung gefunden
\item {\bfseries{Statusanzeige}} und {\bfseries{Nicht-\/stören-\/\+Modus}}
\item {\bfseries{Speicherung}} und {\bfseries{Anzeige}} des gesamten Chatverlaufs
\item {\bfseries{Konfigurations-\/\+Reload}} im laufenden Betrieb möglich
\end{DoxyItemize}\hypertarget{index_autotoc_md4}{}\doxysubsection{\texorpdfstring{Technischer Ansatz}{Technischer Ansatz}}\label{index_autotoc_md4}

\begin{DoxyItemize}
\item {\itshape Peer-\/to-\/\+Peer Architektur}\+: Jeder Nutzer ist gleichberechtigt, es gibt keinen Server
\item {\itshape Kommunikation}\+: UDP-\/\+Broadcast für Discovery, TCP für Messaging/\+Bilder
\item {\itshape Prozessmodell}\+: Jeder Hauptteil (Discovery, Messaging, UI) läuft als separater Prozess
\item {\itshape Prozesssynchronisation}\+: Über Python multiprocessing.\+Manager und Queues
\item {\itshape Modular}\+: Die Software ist modular und rein funktionsbasiert geschrieben, für maximale Nachvollziehbarkeit
\end{DoxyItemize}\hypertarget{index_autotoc_md5}{}\doxysubsection{\texorpdfstring{Bedienung}{Bedienung}}\label{index_autotoc_md5}

\begin{DoxyItemize}
\item Gestartet wird mit {\ttfamily python \doxylink{start_8py}{start.\+py} config1.\+toml} (bzw. mit einer anderen Konfiguration für weitere Nutzer)
\item Im Chat stehen intuitive Befehle wie {\ttfamily msg}, {\ttfamily img}, {\ttfamily who}, {\ttfamily contacts}, {\ttfamily status}, {\ttfamily help} etc. bereit
\item Bei Beenden wird alles sauber geschlossen und der Verlauf kann gespeichert werden

\hypertarget{index_autotoc_md8}{}\doxysubsection{\texorpdfstring{Detaillierter Aufbau}{Detaillierter Aufbau}}\label{index_autotoc_md8}
Plauderkiste besteht aus den folgenden Modulen\+:


\begin{DoxyItemize}
\item {\bfseries{\doxylink{start_8py}{start.\+py}\+:}} Initialisiert alles, startet Prozesse, hält alles zusammen
\item {\bfseries{\doxylink{discovery__comm_8py}{discovery\+\_\+comm.\+py}\+:}} Findet andere Nutzer im Netzwerk (UDP-\/\+Broadcast)
\item {\bfseries{\doxylink{network__comm_8py}{network\+\_\+comm.\+py}\+:}} Übernimmt das Versenden und Empfangen von Nachrichten/\+Bildern (TCP)
\item {\bfseries{\doxylink{ui__cli_8py}{ui\+\_\+cli.\+py}\+:}} Das Kommandozeilen-\/\+Interface für Eingabe, Status und Darstellung
\item {\bfseries{\doxylink{data__manager_8py}{data\+\_\+manager.\+py}\+:}} Speichern und Nachladen des Verlaufs und der Konfiguration
\end{DoxyItemize}\hypertarget{index_autotoc_md9}{}\doxysubsection{\texorpdfstring{Wichtige Designentscheidungen und Herausforderungen}{Wichtige Designentscheidungen und Herausforderungen}}\label{index_autotoc_md9}

\begin{DoxyItemize}
\item {\itshape Synchronisierung ohne Datenverlust}\+: Durch periodisches "{}\+WHO"{}/"{}\+SEEN"{}-\/Verfahren und klare Fehlerbehandlung
\item {\itshape Unabhängige Prozesse}\+: Prozesse laufen unabhängig, Datenaustausch nur über Queues und Manager-\/\+Objekte
\item {\itshape Modularität}\+: Jede Funktionalität (Discovery, Netzwerk, UI, Daten) ist ein eigenes Modul – verständlich für neue Entwickler
\item {\itshape Benutzerfreundlichkeit}\+: Einfache, farbige CLI mit klaren Kommandos und Fehlerausgaben
\end{DoxyItemize}\hypertarget{index_autotoc_md10}{}\doxysubsection{\texorpdfstring{Bekannte Probleme \& Hinweise}{Bekannte Probleme \& Hinweise}}\label{index_autotoc_md10}
\hypertarget{index_autotoc_md11}{}\doxysubsubsection{\texorpdfstring{Windows-\/spezifische Einschränkungen}{Windows-\/spezifische Einschränkungen}}\label{index_autotoc_md11}

\begin{DoxyItemize}
\item {\bfseries{UDP-\/\+Broadcast auf localhost}}\+: Unter Windows funktioniert UDP-\/\+Broadcast mit mehreren Instanzen auf derselben Maschine nur eingeschränkt. Wenn Sie mehrere Clients auf einem Gerät starten, kann es sein, dass nicht alle sich gegenseitig sehen. Im realen Netzwerk (mehrere Geräte) oder unter Linux/\+Mac gibt es dieses Problem nicht.
\item {\bfseries{Firewall blockiert Kommunikation}}\+: Windows-\/\+Firewall (oder Antivirus) kann sowohl UDP-\/ als auch TCP-\/\+Verbindungen blockieren. Geben Sie im Zweifel dem Python-\/\+Interpreter (z.\+B. python.\+exe) Zugriffsrechte für das lokale Netzwerk.
\item {\bfseries{Port bereits belegt}}\+: Falls beim Starten die Fehlermeldung {\ttfamily OSError\+: \mbox{[}Win\+Error 10048\mbox{]}} erscheint, ist der gewünschte Port schon in Benutzung. Ändern Sie in der jeweiligen Konfigurationsdatei ({\ttfamily config\+X.\+toml}) den Wert für {\ttfamily port} und {\ttfamily whoisport} auf einen anderen, noch freien Port.
\end{DoxyItemize}\hypertarget{index_autotoc_md12}{}\doxysubsubsection{\texorpdfstring{Allgemeine Hinweise}{Allgemeine Hinweise}}\label{index_autotoc_md12}

\begin{DoxyItemize}
\item {\bfseries{Images werden nicht immer automatisch angezeigt}}\+: Das Öffnen empfangener Bilder funktioniert nur, wenn das Betriebssystem das zugehörige Standardprogramm richtig hinterlegt hat.
\item {\bfseries{Verlorene Nachrichten bei Netzwerkproblemen}}\+: Da Discovery über UDP läuft, kann es in seltenen Fällen passieren, dass Nachrichten zur Nutzererkennung (WHO/\+SEEN) verloren gehen und temporär nicht alle Nutzer angezeigt werden. Ein erneutes Ausführen von {\ttfamily who} löst das Problem meist.
\item {\bfseries{Erreichbarkeit im WLAN/\+Netzwerk}}\+: In manchen WLANs blockiert der Router lokale Broadcasts. Prüfe ggf. deine Router-\/\+Einstellungen oder wechsel in ein anderes Netzwerk.
\end{DoxyItemize}

Diese Einschränkungen liegen {\bfseries{außerhalb des Einflusses der Software}} und betreffen grundlegende Eigenschaften von Betriebssystemen, Netzwerken oder Geräteeinstellungen.
